Design rules — non-negotiable
R01context.accent NEVER on ring arc, story text, or CTA when urgencyLevel is HIGH or MEDIUMcritical
R02Urgency colours NEVER in context type badge or card cover strip
R03Funding bar fill is ALWAYS context.accent — never urgency, regardless of health state
R04Budget bar switches to urgency.high.fill (#E24B4A) when pct_used ≥ 85
R05REMIND_OVERDUE CTA is ALWAYS gradient.urgency.cta (teal #10B981) — never context accentcritical
R06All CTA text is #FFFFFF — every context accent is dark enough, no exceptions
R07base.bg #120F20 is the ONLY allowed screen background — no variant, no context overridecritical
R08No colour hex hardcoded in any component — always reference a token
R09PERSONAL hides: participant stack, Remind CTA, social activity copy, Share on completion
R10BUSINESS replaces Alerts → Receipts, disables confetti, sets default split to ROLE_BASED
R11CIRCLE posts NEVER show exact figures — abstracted only (e.g. "Saved ₹50k this year")
R12Gradients appear ONLY on: hero card glow, header gradient, CTA buttons — all other surfaces flat
R13Ring outer track uses context.surface colour — NOT base.s200
R14Plus Jakarta Sans is the ONLY typeface. Fallback: DM Sans, Inter, -apple-system
R15Pill radius (999dp) for: context badge, tab chips, filter chips, type tags, status badges, avatars
R16Group context orb opacity = 0.30 (all other contexts = 0.25) — warmth tweak v2.1v2.1
R17Business context uses Burnished Amber (#D4880A → #F5A623) not blue — thermally opposite to Personal (indigo) and Group (teal), semantically correct for financial audit workv2.2
